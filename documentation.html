01

Overview

BMOS is a lightweight command-line operating environment built for experimentation, customization, packages, and learning. It is designed to feel simple enough to understand, but expandable enough to grow.

Image placeholder: BMOS main screen or terminal preview
BMOS is not only for Raspberry Pi. It can be used wherever your supported BMOS build or environment is designed to run.

02

Versions

BMOS has multiple editions depending on what kind of experience you want.

BMOS 1.2 Core

The recommended version for most users.

BMOS Raw

Experimental preview builds for testing upcoming features.

BMOS Education

A future classroom-focused version with guided learning tools.

03

Requirements

Before installing BMOS, make sure you have the correct files, enough storage, and a supported device or environment.

  • BMOS image or installer file
  • 16GB or larger storage recommended
  • A computer to download and prepare BMOS
  • A keyboard and display for setup
  • Internet access for updates and packages
Recommended storage: 16GB+
Recommended version: BMOS 1.2 Core
Continue to Download BMOS →
04

Download BMOS

Go to the BMOS download page and choose the version you want. Most users should choose BMOS 1.2 Core.

  1. Open the Download page.
  2. Select BMOS 1.2 Core.
  3. Download the file.
  4. If the file is zipped, extract it first.
Image placeholder: Download page screenshot 
Example file:
BMOS-1.2-Core.img.zip
Continue to Flash or Install →
05

Flash or Install

BMOS may be flashed to a drive, installed into a supported environment, or prepared using the method provided for your device.

For image-based installs

  1. Open Raspberry Pi Imager, Balena Etcher, or another flashing tool.
  2. Select the BMOS image file.
  3. Select your target drive.
  4. Write the image.
  5. Safely eject the drive when finished.
Flashing erases the selected drive. Make sure you choose the correct drive.
Continue to First Boot →
06

First Boot

After BMOS is installed or flashed, boot into it. The first boot may take longer because BMOS may create settings, check packages, and prepare the shell. 

Loading BMOS Core...
Checking packages...
Loading settings...
Starting shell...

Once booting is complete, BMOS will show the main shell prompt. 

bmos>

07

Setup

During setup, BMOS may ask you to create a username, password, or basic configuration.

  • Create your BMOS user.
  • Set a password if required.
  • Confirm settings.
  • Restart BMOS if prompted.

08

Commands

BMOS is controlled through commands. Type a command and press Enter. 

help
apps
settings
repo
update
clear
exit

Common commands

  • help — shows available commands
  • apps — lists installed apps or packages
  • settings — opens BMOS settings
  • repo — checks repository information
  • update — checks for updates
  • clear — clears the screen
  • exit — exits the shell
Continue to Packages →
09

Packages

Packages add extra tools, commands, and features to BMOS. 

/opt/bmos/packages/

A package may include command files, app files, settings, or extra resources BMOS can load.

Basic package process

  1. Download a BMOS package.
  2. Place it in the packages folder.
  3. Restart BMOS.
  4. Run apps or help to check it.

10

Updates

Updates may include new commands, bug fixes, package changes, startup improvements, and security improvements. 

bmos> update

If online updates are not available in your build yet, download the newest BMOS version manually from the Download page.

11

Troubleshooting

BMOS will not boot

  • Reflash the BMOS image.
  • Check that your drive or storage works.
  • Make sure your device/environment is supported.

Packages are not loading

  • Check that the package is in the correct folder.
  • Restart BMOS.
  • Make sure the package is built for your BMOS version.

Internet features do not work

  • Check your network connection.
  • Restart BMOS.
  • Try again later if the repo is offline.
Continue to Support →
12

Support

If you need help, check the documentation first. If the issue is still happening, report the problem with as much detail as possible. 

Include:
• BMOS version
• Device or environment
• What you were doing
• Error message
• Screenshots if possible
