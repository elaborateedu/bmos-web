BMOS Documentation.

Everything needed to install, flash, configure, and use BMOS on your Raspberry Pi.

Requirements

Before installing BMOS, make sure you have the following:

  • Raspberry Pi 4, Pi 400, or newer recommended
  • MicroSD card (16GB or larger recommended)
  • MicroSD card reader
  • Computer running Windows, macOS, or Linux
  • Internet connection for downloads and updates

BMOS is designed around Raspberry Pi OS Lite and a custom Python shell environment.

Downloading BMOS

BMOS can be downloaded directly from the official BMOS website or GitHub releases page.

Download the latest .img file or compressed archive. 

BMOS-1.2.img.zip

If the file is split into multiple parts: 

BMOS.part1.zip
BMOS.part2.zip
BMOS.part3.zip

Download all parts into the same folder before extracting.

Flashing BMOS

BMOS must be flashed onto a microSD card before booting.


Recommended flashing software:

  • Raspberry Pi Imager
  • Balena Etcher
  • Win32DiskImager

Flashing using Raspberry Pi Imager:

  1. Insert your microSD card into your computer
  2. Open Raspberry Pi Imager
  3. Select "Use Custom"
  4. Select the BMOS .img file
  5. Select your SD card
  6. Press "Write"
  7. Wait for flashing to complete

Once flashing is complete, safely eject the SD card.

First Boot

Insert the flashed SD card into your Raspberry Pi and power it on.


During first boot, BMOS may:

  • Resize the filesystem
  • Initialize packages
  • Create configuration files
  • Run setup and password creation

You may see startup messages similar to: 

Loading BMOS Core...
Checking packages...
Initializing settings...
Starting shell...

Once complete, the BMOS shell will launch automatically.

Commands

BMOS includes a command-based shell environment.


Common commands:

help
apps
settings
clear
repo
update
exit

Example:

bmos> help

This displays a list of available BMOS commands.

Packages

BMOS supports package-style applications.


Packages are typically installed as zip files: 

weather.zip
notes.zip
music.zip

Installed packages are stored in: 

/opt/bmos/packages/

BMOS automatically scans this folder during startup.


Installing a package manually:

  1. Download the package zip
  2. Move it into the packages folder
  3. Restart BMOS

Updates

BMOS updates may include:

  • New shell commands
  • Performance improvements
  • Package manager updates
  • Security fixes

Future versions of BMOS may support online updating directly from repositories.

Repository

BMOS repositories contain package listings and update information.


Example repo.json:

{
  "packages": [
    {
      "name": "weather",
      "version": "1.0",
      "file": "weather.zip"
    },
    {
      "name": "notes",
      "version": "1.2",
      "file": "notes.zip"
    }
  ]
}

BMOS reads this file to determine available downloads and updates.

Troubleshooting

BMOS does not boot:

  • Reflash the image
  • Verify the SD card is working properly
  • Use a Raspberry Pi 4 or newer if possible

Packages are not loading:

  • Verify the package is a valid zip file
  • Place the file in the correct packages directory
  • Restart BMOS

Permission denied errors:

sudo chmod +x filename

System updates fail:

  • Check internet connection
  • Verify repository configuration
  • Restart BMOS and try again

Additional support and future documentation will be available through the BMOS website and GitHub repository.
